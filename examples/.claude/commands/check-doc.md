**$1** のドキュメントを読み込み、以下の項目をチェックしてください：

## チェック項目

### 📋 必須セクション

- [ ] **ドキュメント情報セクション**が存在する
  - 作成日
  - 最終更新日
  - バージョン
  - 作成者
  - プロジェクト名
  - ステータス

- [ ] **目次**が存在する

- [ ] **概要セクション**で目的と範囲が明確に説明されている

- [ ] **変更履歴セクション**がドキュメント末尾に存在する

### 📐 構造とフォーマット

- [ ] **見出し階層**が正しい
  - H1（#）は1回のみ使用
  - H2→H3→H4の順で階層化
  - 見出しレベルをスキップしていない

- [ ] **リスト**が適切にフォーマットされている
  - インデントが統一されている
  - 箇条書きと番号付きリストが適切に使い分けられている

### 💻 コードとMermaid

- [ ] **コードブロック**に言語指定がある
  ```typescript  # ✅ Good
  ```           # ❌ Bad
  ```

- [ ] **Mermaid図**の構文が正しい
  - 開始・終了タグが正しい
  - 構文エラーがない
  - 日本語ラベルが適切に使われている

### 🔗 リンクと参照

- [ ] **内部リンク**が有効
  - 見出しへのアンカーリンクが正しい
  - 相対パスが正しい

- [ ] **外部リンク**が有効（可能な範囲で確認）

- [ ] **関連ドキュメント**へのリンクが設定されている

### ✍️ 文章品質

- [ ] **誤字脱字**がない

- [ ] **専門用語**が統一されている

- [ ] **日本語**が適切に使用されている

- [ ] **具体例**が含まれている（必要に応じて）

### 🔢 メタデータ

- [ ] **バージョン番号**がセマンティックバージョニング形式
  - 例: 1.0.0, 2.1.3

- [ ] **日付**がYYYY-MM-DD形式

- [ ] **ステータス**が適切
  - Draft（下書き）
  - Review（レビュー中）
  - Approved（承認済み）
  - Deprecated（非推奨）

## チェック結果レポート

チェック完了後、以下の形式でレポートを出力：

```markdown
# ドキュメント品質チェック結果

📄 **ファイル**: $1
📅 **チェック日**: YYYY-MM-DD

## 総合評価
[✅ 合格 / ⚠️ 要改善 / ❌ 不合格]

## 詳細結果

### ✅ 良好な点
- [良好な点1]
- [良好な点2]

### ⚠️ 改善推奨
- [改善点1]: [具体的な提案]
- [改善点2]: [具体的な提案]

### ❌ 修正必須
- [問題点1]: [修正方法]
- [問題点2]: [修正方法]

## 修正提案

[具体的な修正案があれば提示]
```

## 実行例

```
/check-doc docs/01_planning/project_overview.md
```

結果：
```

📅 **チェック日**: 2025-01-15

## 総合評価
⚠️ 要改善

## 詳細結果

### ✅ 良好な点

- 見出し階層が適切に整理されている
- Mermaid図が効果的に使用されている

- なし
