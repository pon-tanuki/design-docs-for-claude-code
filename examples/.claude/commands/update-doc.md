以下の手順で **$1** のドキュメントを更新してください：

## 更新手順

### 1. ファイルの読み込み
- $1 のファイルを読み込む

### 2. 内容の更新
- $2 に基づいて内容を更新する
- 既存の構造とフォーマットを維持する

### 3. メタデータの自動更新

#### ドキュメント情報セクション
```markdown
- **最終更新日**: YYYY-MM-DD  # <- 今日の日付に更新
```

#### 変更履歴セクション（ドキュメント末尾）
新しい行を追加：
```markdown
| バージョン | 日付 | 変更者 | 変更内容 |
|-----------|------|--------|----------|
| X.Y.Z     | YYYY-MM-DD | [変更者名] | $2 の内容 |
```

**バージョニングルール**:
- 大幅な変更・破壊的変更 → メジャーバージョンアップ (1.0.0 → 2.0.0)
- 機能追加・セクション追加 → マイナーバージョンアップ (1.0.0 → 1.1.0)
- 軽微な修正・誤字訂正 → パッチバージョンアップ (1.0.0 → 1.0.1)

### 4. 品質チェック

更新後、以下を確認：
- [ ] 見出し階層が正しい（H1は1回のみ）
- [ ] コードブロックに言語指定がある
- [ ] Mermaid図の構文が正しい
- [ ] リンクが有効である
- [ ] 最終更新日が今日の日付になっている
- [ ] 変更履歴に新しいエントリが追加されている

### 5. 変更内容のサマリー

更新完了後、以下を表示：
```
✅ ドキュメント更新完了

📄 ファイル: $1
📝 変更内容: $2
📅 更新日: YYYY-MM-DD
🔢 バージョン: X.Y.Z

主な変更点:
- [具体的な変更内容1]
- [具体的な変更内容2]
```

## 注意事項

- ドキュメントの既存フォーマットを尊重する
- 不要な空行や改行を追加しない
- 他のセクションに影響を与えないよう注意する
- Mermaid図を変更する場合は構文エラーに注意
